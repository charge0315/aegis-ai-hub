# Evaluation Rubric: Aegis AI Hub (Autonomous Evolution)

以下の基準に基づき、実装された自律型ガジェットダッシュボードを評価してください。

## 1. AI サイト発見機能 (Weight: 0.3)
- [ ] Gemini APIに対し、`interests.json`に基づいた適切なプロンプトが投げられているか。
- [ ] 提案されたURLに対して、実際に`RSSFetcher`等を用いて疎通確認（RSSパース可能性の検証）を行っているか。
- [ ] 検証を通過したサイトが正しく`feed_config.json`（またはDB）に反映されているか。
- [ ] 既存のフィードとの重複チェックが行われているか。

## 2. 自律進化サイクル (Weight: 0.3)
- [ ] 定期実行ジョブ（または手動トリガー可能な同等の機能）が実装されているか。
- [ ] 稼働率の低い（エラーが多い）フィードを自動的に削除、または`pool`から別のフィードへ差し替えているか。
- [ ] `interests.json`内のキーワード整理（重複削除や正規化）が行われているか。
- [ ] 取得済み記事から新しいトレンドを抽出し、ユーザーへの「提案リスト」を作成しているか。

## 3. 技術的堅牢性とコード品質 (Weight: 0.2)
- [ ] 設定ファイルの書き込みに `write-file-atomic` 等を使用し、破損を防止しているか。
- [ ] 非同期処理（Gemini API、RSSフェッチ）のエラーハンドリングが適切か。
- [ ] 既存の `server/src/services/` 構造を活かしつつ、関心の分離（Separation of Concerns）がなされているか。
- [ ] Geminiのプロンプトが具体的で、期待する形式（JSON等）を安定して返せるよう工夫されているか。

## 4. UI/UX と進化の可視化 (Weight: 0.2)
- [ ] AIが「いつ、どのサイトを追加し、なぜその興味を提案したか」というログやステータスがユーザーに表示されているか。
- [ ] 新しい興味の提案に対し、ユーザーが「承認/却下」を行うインターフェースが存在するか。
- [ ] 全体的なデザインが「先進的な自律システム」を感じさせる洗練されたもの（ダークテーマ、ネオンアクセント等）になっているか。

## 評価ランク基準
- **S**: 全てのMust/Should項目を満たし、UIが極めて洗練されており、自律進化のログが詳細かつ直感的。
- **A**: 全てのMust項目と大部分のShould項目を満たし、安定して動作している。
- **B**: Must項目を満たしているが、UIのフィードバックが不足しているか、エラーハンドリングに一部甘さがある。
- **C**: 主要な機能（サイト発見、自動進化）の一部が動作しない。
